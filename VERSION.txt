=== SECURITY FIXES (v1.4.6 -> v1.5.1) ===

C-01 (CRITICAL) Governance snapshot bypass
  Files: crates/governance/src/proposal.rs
  Change: Validators NOT in snapshot get ZERO vote weight.
         Previously fell through to live stake, enabling governance takeover.
  Verified by: OpenZeppelin, Halborn (independent confirmation)

C-02 (CRITICAL) Event::decode() OOM via unbounded bincode
  File: crates/hashgraph/src/event.rs
  Change: bincode::options().with_limit(MAX_PAYLOAD_SIZE + 4096)
         + post-decode payload size check. Prevents remote OOM crash.
  Verified by: Trail of Bits, CertiK, Halborn (3x independent confirmation)

C-02b (CRITICAL) Checkpoint::decode() OOM
  File: crates/sync/src/checkpoint.rs
  Change: bincode size limit 256 MiB + pre-check bytes.len().
  Verified by: Cyfrin, Halborn

C-03 (CRITICAL) Merkle tree domain separation (RFC 6962)
  Files: crates/crypto/src/hash.rs, crates/crypto/src/merkle.rs
  Change: 0x01 prefix for internal nodes, 0x00 prefix for leaves.
         New leaf_hash() function. Prevents second-preimage attacks.
  Verified by: CertiK, Halborn

C-04 (CRITICAL) TX hash canonical bincode encoding
  File: crates/types/src/transaction.rs
  Change: bincode::options().with_fixint_encoding().with_big_endian()
         Prevents consensus fork on bincode version upgrade.
  Verified by: CertiK, Spearbit

C-06 (CRITICAL) Global transfer_lock -> per-address ordered locking
  File: crates/executor/src/state.rs
  Change: DashMap<Address, Arc<Mutex<()>>> with deterministic ordering
         (smaller address locked first). Deadlock-free, parallel transfers.
  Verified by: Spearbit, Halborn

OZ-002 (CRITICAL) deactivate() access control
  File: crates/governance/src/validator.rs
  Change: Added caller parameter, only self-deactivation allowed.
  Verified by: OpenZeppelin, Halborn

OZ-005 (HIGH) Quorum 2/3 threshold precision
  File: crates/governance/src/proposal.rs
  Change: votes*3 > total*2 instead of votes > total*2/3
         Eliminates integer division precision loss.
  Verified by: OpenZeppelin

OZ-006 (HIGH) Block active validator re-registration
  File: crates/governance/src/validator.rs
  Change: All re-registration blocked (active or deactivated).
  Verified by: OpenZeppelin, Halborn

OZ-003 (HIGH) voting_deadline overflow protection
  File: crates/governance/src/proposal.rs
  Change: saturating_add instead of raw addition.
  Verified by: OpenZeppelin

OZ-004 (HIGH) Proposal limit (DoS prevention)
  File: crates/governance/src/proposal.rs
  Change: MAX_ACTIVE_PROPOSALS = 128.
  Verified by: OpenZeppelin

OZ-011 (MEDIUM) Endpoint control character rejection
  File: crates/governance/src/validator.rs
  Change: Reject bytes < 0x20 and DEL (0x7F) in endpoint strings.
  Verified by: OpenZeppelin

SP-001 (HIGH) Mempool chain_id validation
  Files: crates/mempool/src/lib.rs + 7 callsites
  Change: Reject transactions for wrong chain before pool insertion.
  Verified by: Spearbit, Halborn

CK-005/006 (HIGH) Constant-time PartialEq on Ed25519 types
  File: crates/crypto/src/signature.rs
  Change: Manual PartialEq via subtle::ConstantTimeEq for
         Ed25519PublicKey and Ed25519Signature.
  Verified by: CertiK, Halborn

CK-012 (MEDIUM) Event hash domain separation tag
  File: crates/crypto/src/hash.rs
  Change: "cathode-event-v1:" prefix in event_id() hash.
  Verified by: Trail of Bits, CertiK

CK-001 (CRITICAL) Merkle leaf_hash() function
  File: crates/crypto/src/hash.rs
  Change: New leaf_hash() with 0x00 domain prefix.
  Verified by: CertiK

TOCTOU (HIGH) Single write lock in vote()
  File: crates/governance/src/proposal.rs
  Change: Eliminated double read+write lock, single write lock from start.
  Verified by: OpenZeppelin
